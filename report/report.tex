\begin{abstract}
Large language models (LLMs) are increasingly used to automate software development, yet most evaluations focus on high-level languages such as Python. This project extends the study by Barati Nia et al.\ \cite{llmgen2025} by evaluating whether state-of-the-art LLMs can generate efficient \textit{Java} implementations of graph algorithms when provided only with C language source files as context. We benchmark three LLMs --- OpenAI ChatGPT, Anthropic Claude, and Google Gemini --- using two approaches: an optimization approach for triangle counting and an algorithm-synthesis approach covering triangle counting, diameter finding, and clique number. \section{Introduction}

The rapid growth of large language models (LLMs) has opened new possibilities for automated software development. Recent work has begun to evaluate not just whether LLM-generated code is correct, but whether it is \textit{efficient} --- particularly for performance-critical applications \cite{llmgen2025, effibench2024}.

A key open question is how LLMs handle cross-language code generation: can a model generate efficient code in one language (Java) when all provided context is written in another (C)? This is a practically important scenario, as many legacy systems are written in C while new development often targets higher-level languages. Evaluating this capability provides insight into LLMs' true algorithmic reasoning ability, separating genuine understanding from simple pattern matching.

This project replicates and extends the experimental evaluation of Barati Nia et al.\ \cite{llmgen2025}, which benchmarked eight LLMs generating C implementations of graph algorithms. We instead prompt three LLMs --- [LLM 1], [LLM 2], and [LLM 3] --- to generate Java implementations while providing only C source files as context, using the same RMAT benchmark graphs \cite{rmat2004} ranging from scale 6 to 18.

We chose these three LLMs because [MOTIVATE YOUR CHOICE HERE --- e.g., they represent the three dominant commercial AI providers, they are publicly accessible, and they scored competitively in the original paper].
